# Design Document

## Overview

CyberArena is a full-stack web and mobile application built on React (frontend), Node.js + Express (backend), and an AI layer for dynamic scenario generation. The architecture is service-oriented, with clearly separated concerns: scenario delivery, adaptive logic, gamification, progress persistence, and accessibility. All services communicate over REST APIs. The data layer uses Firebase for real-time user state and MongoDB for analytics and scenario content.

## Component Design

### 1. Auth Service

Handles registration, login, session management, and SSO.

```
POST /auth/register     → create user, hash password (bcrypt), return JWT
POST /auth/login        → validate credentials, return JWT (24h expiry)
POST /auth/sso          → validate SSO token, return JWT
POST /auth/refresh      → refresh JWT before expiry
GET  /auth/me           → return current user profile
```

- Passwords hashed with bcrypt (cost factor 12)
- JWT signed with RS256, 24-hour expiry, refresh token stored in HttpOnly cookie
- SSO via OAuth 2.0 / SAML 2.0 for enterprise integrations

### 2. Scenario Engine

Selects, delivers, and evaluates scenarios per mission type.

```
GET  /scenarios/missions              → list all four missions
GET  /scenarios/missions/:id          → mission detail + user progress
GET  /scenarios/:id                   → fetch single scenario
POST /scenarios/:id/decision          → submit user decision, get outcome
GET  /scenarios/resume/:userId        → get last incomplete scenario
```

Scenario data model:
```json
{
  "id": "string",
  "missionType": "phishing | social_engineering | ai_crime | malware",
  "difficulty": "beginner | intermediate | advanced",
  "content": { ... },
  "decisions": [{ "id": "string", "label": "string", "outcomeId": "string" }],
  "outcomes": [{ "id": "string", "consequence": "string", "isCorrect": boolean, "feedbackId": "string" }],
  "prerequisites": ["scenarioId"]
}
```

### 3. Adaptive Engine

Tracks per-user performance and adjusts difficulty queuing.

```
POST /adaptive/record        → record attempt result (userId, scenarioId, correct)
GET  /adaptive/next/:userId  → return next recommended scenario + difficulty
GET  /adaptive/summary/:userId → return weakness summary + refresher list
```

Adaptive logic:
- Maintain a rolling window of the last 10 attempts per mission type per user
- Correct rate < 50% → queue more scenarios at same difficulty
- Correct rate > 80% over 3 consecutive → promote to next difficulty
- Weakness summary generated by ranking mission types by lowest correct rate

### 4. AI Layer

Generates scenario content on demand or in batch for the scenario pool.

```
POST /ai/generate/phishing        → generate phishing email sample
POST /ai/generate/deepfake-text   → generate AI-crafted scam text
POST /ai/generate/social-eng      → generate social engineering script
GET  /ai/generate/status/:jobId   → check async generation job status
```

- Uses an LLM (e.g. GPT-4o) for text generation with structured prompts per difficulty
- Deepfake image samples sourced from a curated dataset with artefact injection at beginner level
- All generated content is reviewed against a safety filter before entering the scenario pool
- Generation target: respond within 2 seconds for synchronous requests; batch pre-generation fills the pool nightly

### 5. Gamification Service

Manages XP, levels, badges, streaks, and leaderboards.

```
POST /gamification/award-xp        → award XP for scenario completion
GET  /gamification/profile/:userId → XP, level, streak, badges
GET  /gamification/leaderboard/:teamId → ranked team leaderboard
POST /gamification/check-badges    → evaluate and award new badges
```

XP and level table (example):
| Level | XP Required |
|-------|-------------|
| 1     | 0           |
| 2     | 100         |
| 3     | 300         |
| 4     | 600         |
| ...   | ...         |

XP per scenario by difficulty:
- Beginner: 10 XP
- Intermediate: 25 XP
- Advanced: 50 XP

Badge criteria:
| Badge          | Criteria                                      |
|----------------|-----------------------------------------------|
| Phishing Pro   | Complete all Phishing Detective scenarios     |
| Scam Shield    | Complete all Social Engineering scenarios     |
| AI Detector    | Complete all AI-Crime Lab scenarios           |
| Escape Artist  | Complete all Malware Escape Room scenarios    |
| Streak Master  | Maintain a 7-day streak                       |

### 6. Feedback Service

Delivers immediate consequence feedback and micro-learning content.

```
GET /feedback/:outcomeId   → return consequence, micro-learning, practical tip
GET /feedback/:outcomeId/expanded → return extended explanation
```

Feedback data model:
```json
{
  "id": "string",
  "consequence": "string (≤ 100 words)",
  "microLearning": "string (≤ 100 words)",
  "practicalTip": "string",
  "expandedExplanation": "string"
}
```

Feedback is pre-authored per outcome and stored in MongoDB. The service returns feedback within 500ms by serving from an in-memory cache (Redis or Node.js LRU cache).

### 7. Progress Store

Persists all user progress, analytics, and metrics.

- Firebase Realtime Database: live user state (current scenario, streak, XP) for low-latency reads
- MongoDB: scenario attempt history, analytics aggregations, admin reports

Collections:
```
users           → profile, auth metadata
attempts        → { userId, scenarioId, decision, correct, timestamp }
progress        → { userId, missionId, completedScenarios[], currentDifficulty }
gamification    → { userId, xp, level, streak, lastActiveDate, badges[] }
organisations   → { adminId, teams[], assignedMissions[] }
```

### 8. Accessibility Service

Provides voice guidance, subtitles, contrast modes, and localisation.

- Voice guidance: Web Speech API (browser) / Expo Speech (mobile) reads scenario text aloud
- Subtitles: all audio/video content includes a `subtitles` field in the scenario content model
- High-contrast mode: CSS custom properties toggled via a user preference flag; contrast ratios validated against WCAG 2.1 AA
- Simplified reading mode: scenario content includes a `simplified` text variant
- Localisation: i18next (web) / i18n (React Native) with JSON translation files per language; minimum 5 languages at launch (English, Spanish, French, Hindi, Mandarin)

### 9. Admin Service

Organisation and team management.

```
POST /admin/teams              → create team
POST /admin/teams/:id/users    → add user to team
POST /admin/teams/:id/missions → assign mandatory mission
GET  /admin/teams/:id/report   → aggregated team performance
GET  /admin/teams/:id/export   → CSV export of team report
```

## Data Flow: Scenario Execution

```
User selects Mission
      │
      ▼
Adaptive Engine → GET /adaptive/next/:userId
      │  returns { scenarioId, difficulty }
      ▼
Scenario Engine → GET /scenarios/:id
      │  returns scenario content
      ▼
Client renders Scenario
      │
User makes decision
      │
      ▼
Scenario Engine → POST /scenarios/:id/decision
      │  returns { outcomeId, correct }
      ▼
Feedback Service → GET /feedback/:outcomeId
      │  returns { consequence, microLearning, practicalTip }
      ▼
Gamification Service → POST /gamification/award-xp (if correct)
      │
      ▼
Adaptive Engine → POST /adaptive/record
      │
      ▼
Progress Store → write attempt record
```

## Security Considerations

- All API endpoints require a valid JWT except `/auth/register` and `/auth/login`
- Rate limiting on auth endpoints: 10 requests per minute per IP
- AI-generated content passes through a content safety filter before storage
- Admin endpoints require a role claim of `admin` in the JWT
- User data is stored with field-level encryption for PII (email, name)
- HTTPS enforced on all endpoints; HSTS header set

## Correctness Properties

### Property 1: Adaptive Engine Monotonicity
WHEN a User's correct decision rate exceeds 80% over 3 consecutive Scenarios of the same type, THE Adaptive_Engine SHALL never decrease the difficulty level for that type in the same session.

### Property 2: XP Accumulation Consistency
FOR ALL Scenario completions, the User's cumulative XP in the Progress_Store SHALL equal the sum of XP awarded by the Gamification_Service across all recorded attempts.

### Property 3: Feedback Delivery Invariant
FOR ALL User decisions submitted to the Scenario_Engine, THE Feedback_Service SHALL return a feedback object containing all three fields: consequence, microLearning, and practicalTip.

### Property 4: Streak Reset Correctness
IF a User does not record a Scenario attempt on a given calendar day, THEN the streak counter stored in the Progress_Store SHALL be zero on the following day, regardless of prior streak length.

### Property 5: Scenario Prerequisite Enforcement
FOR ALL Scenarios with a non-empty prerequisites list, THE Scenario_Engine SHALL never deliver that Scenario to a User who has not completed all prerequisite Scenarios.

### Property 6: Badge Idempotence
FOR ALL badge award operations, awarding the same badge to the same User multiple times SHALL result in the User holding exactly one instance of that badge.

### Property 7: Leaderboard Consistency
FOR ALL team leaderboard queries, the ranked order SHALL be consistent with the descending order of cumulative XP values stored in the Progress_Store for Users in that team.
